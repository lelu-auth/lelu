# Deploy Lelu to Render

This guide will help you deploy Lelu to Render with automatic CI/CD from GitHub.

## Prerequisites

1. A [Render account](https://render.com) (free tier available)
2. Your GitHub repository connected to Render
3. Docker support enabled

## Quick Deploy

### Option 1: Deploy via Render Dashboard (Recommended)

1. **Fork or push this repository to GitHub**

2. **Go to Render Dashboard**
   - Visit https://dashboard.render.com
   - Click "New +" → "Blueprint"

3. **Connect Repository**
   - Select your GitHub repository
   - Render will automatically detect `render.yaml`

4. **Configure Services**
   - Render will create:
     - `lelu-ui` - Web dashboard (https://lelu-ui.onrender.com)
     - `lelu-postgres` - PostgreSQL database
   
5. **Deploy**
   - Click "Apply"
   - Wait for services to build and deploy (5-10 minutes)

### Option 2: Deploy Individual Services

If you prefer to deploy services separately:

#### 1. Deploy PostgreSQL Database

```bash
# In Render Dashboard:
# New + → PostgreSQL
# Name: lelu-postgres
# Database: lelu
# User: lelu
# Plan: Free
```

#### 2. Deploy Lelu UI

```bash
# In Render Dashboard:
# New + → Web Service
# Connect your repository
# Settings:
```

- **Name**: `lelu-ui`
- **Environment**: Docker
- **Dockerfile Path**: `./platform/ui/Dockerfile`
- **Docker Context**: `./platform/ui`
- **Plan**: Starter (Free)

**Environment Variables**:
```
NODE_ENV=production
PORT=3000
PLATFORM_URL=https://lelu-platform.onrender.com
LELU_ENGINE_URL=https://lelu-engine.onrender.com
PLATFORM_API_KEY=<generate-secure-key>
LELU_API_KEY=<generate-secure-key>
```

## Environment Variables

Generate secure keys using:
```bash
openssl rand -base64 32
```

### Required Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | PostgreSQL connection string | Auto-generated by Render |
| `PLATFORM_API_KEY` | API key for platform | Generate with openssl |
| `LELU_API_KEY` | API key for engine | Generate with openssl |
| `JWT_SIGNING_KEY` | JWT signing secret | Generate with openssl |

## Automatic CI/CD

Once deployed, Render automatically:
- ✅ Builds on every push to `main` branch
- ✅ Runs health checks
- ✅ Auto-deploys if build succeeds
- ✅ Rolls back on failure

## Custom Domain

To use a custom domain (e.g., lelu-ai.com):

1. Go to your service in Render Dashboard
2. Click "Settings" → "Custom Domain"
3. Add your domain: `lelu-ai.com`
4. Update your DNS records:
   ```
   Type: CNAME
   Name: @
   Value: <your-render-url>.onrender.com
   ```

## Monitoring

### Health Checks

All services include health check endpoints:
- UI: `GET /`
- Engine: `GET /healthz`
- Platform: `GET /healthz`

### Logs

View logs in Render Dashboard:
```
Dashboard → Your Service → Logs
```

Or use Render CLI:
```bash
render logs -s lelu-ui
```

## Scaling

### Free Tier Limitations
- Services sleep after 15 minutes of inactivity
- 750 hours/month of runtime
- 512 MB RAM per service

For production use, upgrade to:

## Troubleshooting

### Build Failures

Check build logs:
```bash
Dashboard → Service → Events → View Build Logs
```

Common issues:
- **Docker build timeout**: Increase build timeout in settings
- **Out of memory**: Upgrade to larger plan
- **Missing dependencies**: Check Dockerfile

### Service Not Starting

1. Check environment variables are set correctly
2. Verify health check endpoint is responding
3. Check service logs for errors

### Database Connection Issues

Ensure `DATABASE_URL` is correctly set:
```bash
postgres://user:password@host:5432/database
```
